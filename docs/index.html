Network programming over Tailscale tsnet

Simple TCP, HTTP, UDP, WebSocket, and QUIC APIs for your private mesh.

Truffle gives Node.js apps a small set of familiar networking APIs that run across devices on the same Tailscale tailnet. Start one mesh node, then open sockets, serve HTTP, send datagrams, stream QUIC, exchange messages, sync state, or transfer files without building a coordinator service.

Start with the API Read the reference
mesh.ts 
import { createMeshNode } from '@vibecook/truffle';

const mesh = await createMeshNode({
  appId: 'field-tools',
  deviceName: 'alice-laptop',
  onAuthRequired: (url) => {
    console.log('Open this Tailscale auth URL:', url);
  },
});

const server = mesh.net.createServer((socket) => {
  socket.write('hello from ' + mesh.getLocalInfo().deviceName + '\n');
  socket.pipe(socket);
});

server.listen(7000);

const client = mesh.net.connect({ host: 'bob-laptop', port: 7000 });
client.on('data', (chunk) => console.log(chunk.toString()));
client.write('ping\n');

Quick start

Create one mesh node, then use the protocol you already know.

The JavaScript package resolves the native addon and sidecar for you. On first run, Truffle opens or reports a Tailscale auth URL, then keeps a tsnet-backed node running inside your process.

1. Install

Add the package

Use Node.js 18 or newer. Add @vibecook/truffle. Add ws only if you use the WebSocket namespace. 

pnpm add @vibecook/truffle
pnpm add ws # optional, only for mesh.ws
2. Start

Join an app-scoped mesh

The appId is the isolation boundary. Devices with different app IDs do not see each other as Truffle peers, even on the same tailnet. 

const mesh = await createMeshNode({
  appId: 'notes',
  deviceName: 'alice-laptop',
  autoAuth: true,
});

console.log(mesh.getLocalInfo());
console.table(await mesh.getPeers());
Peer references can be stable device IDs, unique device ID prefixes, device names, Tailscale hostnames, or Tailscale IPs. Names are convenient for examples; use device IDs when your app needs stable identity.

Transport recipes

The core value: familiar network APIs on top of tsnet.

These examples are intentionally small. Run the server side on one device and the client side on another device using the same appId.

TCP

Raw streams with a Node-style server and socket

mesh.net mirrors the useful parts of node:net. Sockets are standard stream.Duplex objects, so existing stream code usually ports by replacing the dial/listen layer. 

// Bob: listen on the mesh
const server = mesh.net.createServer((socket) => {
  socket.on('data', (chunk) => {
    socket.write(Buffer.from('echo: ' + chunk.toString()));
  });
});
server.listen(7000);

// Alice: connect to Bob by device name, ID, hostname, or IP
const socket = mesh.net.connect({ host: 'bob-laptop', port: 7000 });
socket.on('data', (chunk) => console.log(chunk.toString()));
socket.write('hello over private TCP');
HTTP

Serve ordinary node:http traffic over the mesh

Inbound HTTP is a stock http.Server fed by mesh TCP sockets. Outbound HTTP uses mesh.http.agent or fetchText(). 

import http from 'node:http';

// Bob: serve HTTP on mesh port 8080
const app = http.createServer((req, res) => {
  res.end(JSON.stringify({ ok: true, path: req.url }));
});
mesh.net
  .createServer((socket) => app.emit('connection', socket))
  .listen(8080);

// Alice: request Bob over mesh TCP
const res = await mesh.http.fetchText('bob-laptop', 8080, '/status');
console.log(res.status, res.body);
UDP

Datagrams for telemetry, presence, and lightweight signals

mesh.dgram keeps datagram boundaries and emits Node-style message events. Keep payloads small; about 1,200 bytes is the practical target for tailnet MTU. 

// Bob: receive datagrams
const udp = mesh.dgram.createSocket();
udp.on('message', (msg, rinfo) => {
  console.log('from', rinfo.peerName ?? rinfo.address, msg.toString());
});
await udp.bind(9999);

// Alice: send a datagram
const sender = mesh.dgram.createSocket();
await sender.bind();
sender.send(Buffer.from('temperature=21.7'), 9999, 'bob-laptop');
WebSocket

Use the ws package across the private mesh

mesh.ws runs the standard ws package over Truffle TCP. It is an optional peer dependency, so install ws in apps that use it. 

// Bob: WebSocket server
const wss = await mesh.ws.createServer({ port: 7788, path: '/chat' });
wss.on('connection', (ws) => {
  ws.on('message', (data) => ws.send('echo: ' + data.toString()));
});

// Alice: WebSocket client
const ws = await mesh.ws.connect('bob-laptop', 7788, '/chat');
ws.once('open', () => ws.send('hello websocket'));
ws.on('message', (data) => console.log(data.toString()));
QUIC

Multiplexed streams without head-of-line blocking

mesh.quic is useful when one peer needs many concurrent reliable streams over a single connection. Connections and incoming streams are async iterable. 

// Bob: echo every stream on every connection
const quicServer = await mesh.quic.listen(9420);
(async () => {
  for await (const conn of quicServer) {
    (async () => {
      for await (const stream of conn.streams()) {
        stream.pipe(stream);
      }
    })();
  }
})();

// Alice: open a connection and one stream
const conn = await mesh.quic.connect('bob-laptop', 9420);
const stream = await conn.openStream();
stream.on('data', (chunk) => console.log(chunk.toString()));
stream.write('hello over quic');
stream.end();

Higher-level APIs

Messaging, files, and state sync are built on the same mesh node.

Use raw transports when you want protocol control. Use these higher-level helpers when you want common local-first app workflows with less protocol code.

Messages

Namespaced app messages

Route JSON or byte payloads by namespace and subscribe with onMessage(namespace, callback).

Files

Verified file transfer

Send, pull, accept, reject, and track progress. Transfers include SHA-256 verification.

State

Device-owned synced stores

Each device publishes its own versioned JSON slice. Read local and remote slices by device ID.

Messages

The payload is app-defined. If you pass JSON bytes, Truffle delivers a structured JSON value to receivers. 

mesh.onMessage('chat', (msg) => {
  console.log(msg.from, msg.payload.text);
});

const peer = (await mesh.getPeers())[0];
await mesh.send(
  peer.deviceId,
  'chat',
  Buffer.from(JSON.stringify({ text: 'hello' })),
);

File transfer

For push transfers, the receiver can manually accept offers or use autoAccept(). Pull transfers are deny-by-default until roots are added. 

const files = mesh.fileTransfer();
files.onEvent((event) => console.log(event.eventType, event.progress));
files.onOffer((offer, responder) => {
  responder.accept('/tmp/' + offer.fileName);
});

await files.sendFile(peer.deviceId, './report.pdf', '/tmp/report.pdf');

Synced store

Stores are scoped by ID. Each device owns one slice per store and can observe changes from peers. 

const presence = mesh.syncedStore('presence');
presence.onChange((event) => console.log(event));

await presence.set({ status: 'editing', file: 'notes.md' });
const allPresence = await presence.all();
console.table(allPresence);

Companion tools

The CLI is useful, but the API is the product surface.

The truffle command can bring up a node, inspect peers, copy files, stream events, and debug connectivity. It is helpful for development and operations; apps should build directly on the API.

Install

Install the CLI

The CLI is useful for bringing up a node, checking peers, sending quick messages, copying files, and debugging connectivity while you build against the API. 

# macOS / Linux
curl -fsSL https://jamesyong-42.github.io/truffle/install.sh | sh

# Windows
iwr -useb https://jamesyong-42.github.io/truffle/install.ps1 | iex

# Homebrew
brew install jamesyong-42/tap/truffle
Quick start

Run common mesh tasks

These commands use the same mesh concepts as the API: start a node, discover peers, send messages, copy files, and check connectivity. 

# Start your node and join the mesh
truffle up

# See all nodes on your network
truffle ls

# Send a message to another node
truffle send laptop "deploy is done"

# Copy a file to another node
truffle cp ./report.pdf server:/tmp/

# Check connectivity
truffle ping laptop
# Extra commands for development and automation
truffle status --json
truffle watch --json
truffle wait laptop --timeout 30
truffle recv --from laptop --timeout 30
truffle doctor
