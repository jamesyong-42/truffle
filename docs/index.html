Network programming over Tailscale tsnet

Simple TCP, HTTP, UDP, WebSocket, and QUIC APIs for your private mesh.

Truffle gives Node.js apps a small set of familiar networking APIs that run across devices on the same Tailscale tailnet. Start one mesh node, get Peer handles, then open sockets, serve HTTP, send datagrams, stream QUIC, exchange messages, sync state, or transfer files — without building a coordinator or picking id strings.

mesh.ts 
import { createMeshNode } from '@vibecook/truffle';

const mesh = await createMeshNode({
  appId: 'field-tools',
  deviceName: 'alice-laptop',
  onAuthRequired: (url) => {
    console.log('Open this Tailscale auth URL:', url);
  },
});

// Peer handles — no id picking for networking
mesh.onMessage('chat', async (msg) => {
  if (typeof msg.from === 'string') return;
  console.log(msg.from.displayName, msg.payload);
  await msg.from.send('chat', Buffer.from('ack'));
});

const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
if (bob) await bob.send('chat', Buffer.from('hello'));

Quick start

Create one mesh node. Networking takes Peers.

The JavaScript package resolves the native addon and sidecar for you. On first run, Truffle opens or reports a Tailscale auth URL, then keeps a tsnet-backed node running inside your process. Discovery and messages yield Peer handles — you do not choose which id string to pass.

1. Install

Add the package

Use Node.js 18 or newer. Add @vibecook/truffle. Add ws only if you use the WebSocket namespace. 

pnpm add @vibecook/truffle
pnpm add ws # optional, only for mesh.ws
2. Start

Join an app-scoped mesh

The appId is the isolation boundary. Devices with different app IDs do not see each other as Truffle peers, even on the same tailnet. 

const mesh = await createMeshNode({
  appId: 'notes',
  deviceName: 'alice-laptop',
  autoAuth: true,
});

console.log(mesh.getLocalInfo());
const peers = await mesh.getPeers();
for (const p of peers) {
  console.log(p.displayName, p.online);
}
3. Talk

Send with a Peer handle

Prefer handles from getPeers(), events, or mesh.peer(query). String overloads still work as queries (name, ULID, hostname, IP) — not as a multi-id protocol you must learn. 

mesh.onMessage('chat', async (msg) => {
  if (typeof msg.from === 'string') return; // not yet interned
  console.log(msg.from.displayName, msg.payload);
  await msg.from.send('chat', Buffer.from('ack'));
});

const peer = (await mesh.getPeers())[0];
if (peer) await peer.send('chat', Buffer.from('hello'));
// same: await mesh.send(peer, 'chat', Buffer.from('hello'));

Peers

One handle. Live fields. Identity stays inside Truffle.

A Peer is a process-local handle to a live registry entry. The same object is returned from getPeers(), peer-change events, and inbound message.from when known — so Map<Peer, T> and === work.

Display

Always safe fields

Use displayName, online, ip, hostname, and os for UI. Do not gate send on wsConnected — that is advanced bus state, not liveness.

Lookup

mesh.peer(query)

Resolve a name, ULID (or unique prefix), hostname, Tailscale id, or IP. waitMs blocks until the peer is resolvable (for example after a restart while durable identity is learned). Ambiguous names throw.

Persistence

deviceId only when non-null

deviceId is a real ULID or null — never a Tailscale id in disguise. Save it across restarts when set; rebind with mesh.peer(savedId, { waitMs }). Do not persist peer.ref.

Collections

Map by Peer, not by string

Interned handles update in place as discovery and identity change. A peer that leaves and rejoins gets a new object generation (equals is false). Compare durable deviceId only when you need “same device across restarts.” 

const sessions = new Map(); // Map<Peer, AppSession>

mesh.onPeerChange((ev) => {
  if (ev.type === 'joined' && ev.peer) {
    sessions.set(ev.peer, openSession(ev.peer));
  }
  if (ev.type === 'left' && ev.peer) {
    sessions.delete(ev.peer);
  }
});

// Across restarts — only place durable ids appear in app code
if (peer.deviceId) localStorage.setItem('lastPeer', peer.deviceId);
const id = localStorage.getItem('lastPeer');
const again = id ? await mesh.peer(id, { waitMs: 5000 }) : null;
if (again) await again.send('chat', Buffer.from('welcome back'));

What you do not need to learn

Which string send wants before vs after WebSocket hello; dual-key maps to survive disconnect; that raw-transport apps should key on tailscaleId. Routing keys stay inside the library. tailscaleId remains available for diagnostics only.

Transport recipes

The core value: familiar network APIs on top of tsnet.

These examples are intentionally small. Run the server side on one device and the client side on another device using the same appId. Prefer a Peer from getPeers() or mesh.peer(); string hosts remain valid as queries.

TCP

Raw streams with a Node-style server and socket

mesh.net mirrors the useful parts of node:net. Sockets are standard stream.Duplex objects, so existing stream code usually ports by replacing the dial/listen layer. Pass peer (or host) as a handle. 

// Bob: listen on the mesh
const server = mesh.net.createServer((socket) => {
  socket.on('data', (chunk) => {
    socket.write(Buffer.from('echo: ' + chunk.toString()));
  });
});
server.listen(7000);

// Alice: resolve a Peer, then dial
const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
if (!bob) throw new Error('bob not found');
const socket = mesh.net.connect({ peer: bob, port: 7000 });
// also: mesh.net.connect({ host: bob, port: 7000 })
// or query form: mesh.net.connect({ host: 'bob-laptop', port: 7000 })
socket.on('data', (chunk) => console.log(chunk.toString()));
socket.write('hello over private TCP');
HTTP

Serve ordinary node:http traffic over the mesh

Inbound HTTP is a stock http.Server fed by mesh TCP sockets. Outbound HTTP uses mesh.http.agent or fetchText(). Host may be a Peer or a query string. 

import http from 'node:http';

// Bob: serve HTTP on mesh port 8080
const app = http.createServer((req, res) => {
  res.end(JSON.stringify({ ok: true, path: req.url }));
});
mesh.net
  .createServer((socket) => app.emit('connection', socket))
  .listen(8080);

// Alice: request Bob over mesh TCP (query string; MeshAgent dials via mesh.net)
const res = await mesh.http.fetchText('bob-laptop', 8080, '/status');
console.log(res.status, res.body);

// Prefer a Peer when dialing TCP yourself:
const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
if (bob) mesh.net.connect({ peer: bob, port: 8080 });
UDP

Datagrams for telemetry, presence, and lightweight signals

mesh.dgram keeps datagram boundaries and emits Node-style message events. Keep payloads small; about 1,200 bytes is the practical target for tailnet MTU. The address argument is Peer | string. 

// Bob: receive datagrams
const udp = mesh.dgram.createSocket();
udp.on('message', (msg, rinfo) => {
  console.log('from', rinfo.peerName ?? rinfo.address, msg.toString());
});
await udp.bind(9999);

// Alice: send a datagram to a Peer handle
const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
const sender = mesh.dgram.createSocket();
await sender.bind();
sender.send(Buffer.from('temperature=21.7'), 9999, bob ?? 'bob-laptop');
WebSocket

Use the ws package across the private mesh

mesh.ws runs the standard ws package over Truffle TCP. It is an optional peer dependency, so install ws in apps that use it. Connect with a Peer handle or a query string. 

// Bob: WebSocket server
const wss = await mesh.ws.createServer({ port: 7788, path: '/chat' });
wss.on('connection', (ws) => {
  ws.on('message', (data) => ws.send('echo: ' + data.toString()));
});

// Alice: WebSocket client
const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
const ws = await mesh.ws.connect(bob ?? 'bob-laptop', 7788, '/chat');
ws.once('open', () => ws.send('hello websocket'));
ws.on('message', (data) => console.log(data.toString()));
QUIC

Multiplexed streams without head-of-line blocking

mesh.quic is useful when one peer needs many concurrent reliable streams over a single connection. Connections and incoming streams are async iterable. connect takes Peer | string. 

// Bob: echo every stream on every connection
const quicServer = await mesh.quic.listen(9420);
(async () => {
  for await (const conn of quicServer) {
    (async () => {
      for await (const stream of conn.streams()) {
        stream.pipe(stream);
      }
    })();
  }
})();

// Alice: open a connection and one stream
const bob = await mesh.peer('bob-laptop', { waitMs: 5000 });
const conn = await mesh.quic.connect(bob ?? 'bob-laptop', 9420);
const stream = await conn.openStream();
stream.on('data', (chunk) => console.log(chunk.toString()));
stream.write('hello over quic');
stream.end();

Higher-level APIs

Messaging, files, and state sync are built on the same mesh node.

Use raw transports when you want protocol control. Use these higher-level helpers when you want common local-first app workflows with less protocol code. Peer-addressed surfaces take a handle first.

Messages

Namespaced app messages

Route JSON or byte payloads by namespace. Subscribe with onMessage; reply with msg.from.send(...) when from is a Peer.

Files

Verified file transfer

Send, pull, accept, reject, and track progress. Transfers include SHA-256 verification. Target peers by handle or query string.

State

Device-owned synced stores

Each device publishes its own versioned JSON slice. Slice keys are durable device ULIDs once identity is known.

Messages

The payload is app-defined. If you pass JSON bytes, Truffle delivers a structured JSON value to receivers. Prefer the Peer handle — never dig out deviceId just to call send. 

mesh.onMessage('chat', async (msg) => {
  if (typeof msg.from === 'string') {
    console.log(msg.from, msg.payload); // rare: sender not interned yet
    return;
  }
  console.log(msg.from.displayName, msg.payload);
  await msg.from.send('chat', Buffer.from(JSON.stringify({ text: 'ack' })));
});

const peer = (await mesh.getPeers())[0];
if (peer) {
  await peer.send(
    'chat',
    Buffer.from(JSON.stringify({ text: 'hello' })),
  );
}

File transfer

For push transfers, the receiver can manually accept offers or use autoAccept(). Pull transfers are deny-by-default until roots are added. 

const files = mesh.fileTransfer();
files.onEvent((event) => console.log(event.eventType, event.progress));
files.onOffer((offer, responder) => {
  responder.accept('/tmp/' + offer.fileName);
});

const peer = (await mesh.getPeers())[0];
if (peer) {
  // peerId is a query string (name / ULID / hostname). PeerLike on FT is next.
  await files.sendFile(peer.displayName, './report.pdf', '/tmp/report.pdf');
}

Synced store

Stores are scoped by ID. Each device owns one slice per store and can observe changes from peers. 

const presence = mesh.syncedStore('presence');
presence.onChange((event) => console.log(event));

await presence.set({ status: 'editing', file: 'notes.md' });
const allPresence = await presence.all();
console.table(allPresence);

Companion tools

The CLI is useful, but the API is the product surface.

The truffle command can bring up a node, inspect peers, copy files, stream events, and debug connectivity. It is helpful for development and operations; apps should build directly on the API.

Install

Install the CLI

The CLI is useful for bringing up a node, checking peers, sending quick messages, copying files, and debugging connectivity while you build against the API. 

# macOS / Linux
curl -fsSL https://jamesyong-42.github.io/truffle/install.sh | sh

# Windows
iwr -useb https://jamesyong-42.github.io/truffle/install.ps1 | iex

# Homebrew
brew install jamesyong-42/tap/truffle
Quick start

Run common mesh tasks

These commands use the same mesh concepts as the API: start a node, discover peers, send messages, copy files, and check connectivity. 

# Start your node and join the mesh
truffle up

# See all nodes on your network
truffle ls

# Send a message to another node
truffle send laptop "deploy is done"

# Copy a file to another node
truffle cp ./report.pdf server:/tmp/

# Check connectivity
truffle ping laptop
# Extra commands for development and automation
truffle status --json
truffle watch --json
truffle wait laptop --timeout 30
truffle recv --from laptop --timeout 30
truffle doctor
